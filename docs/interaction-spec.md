# Interaction Specification: No True Man Show

**Version:** 1.0
**Date:** 2026-02-28
**Status:** Approved for implementation

---

## 1. Overview

This document defines how viewers interact with Truman's world. The core philosophy is **gentle nudges**: viewers suggest and influence, but Truman retains agency. All interactions modify the environment -- Truman then observes the changes and reacts.

**Fundamental rule:** Viewer actions never directly control Truman. They change his world. He notices. He decides how to respond.

---

## 2. Interaction Philosophy

### 2.1 Indirection Principle

Every viewer interaction flows through the environment:

```
Viewer action
      |
      v
Environment changes (food appears, weather shifts, letter arrives)
      |
      v
Truman's observation system detects the change
      |
      v
Agent brain decides reaction (may accept, ignore, or refuse)
```

This indirection:
- Maintains the illusion that Truman is unaware of viewers
- Prevents direct manipulation of Truman's behavior
- Creates a layer of narrative interpretation between viewer intent and outcome
- Makes outcomes unpredictable and entertaining

### 2.2 Resistance Mechanic

Truman can refuse viewer suggestions. Refusal rate is **mood-driven** (~30% average):

| Mood State | Refusal Rate | Behavior |
|---|---|---|
| Happy, content | ~15% | Agreeable, open to suggestions |
| Curious, excited | ~20% | Interested but selective |
| Neutral | ~30% | Default refusal rate |
| Bored | ~25% | More willing to try new things |
| Frustrated, anxious | ~45% | More rebellious, contrary |

This creates a visible feedback loop: treat Truman well, and he's more cooperative. Push too hard, and he pushes back.

### 2.3 What Viewers CAN'T Do

- Directly control Truman's movement or speech
- Force Truman to perform unwanted actions
- Access Truman's inner thoughts on demand
- Speed up or slow down time
- Reset Truman's memory
- Override the emotional floor (make Truman suffer)
- Bypass the sanitizer pipeline

## 3. Interaction Tiers

### Tier 1: Chat Votes (Low Impact)

**Purpose:** Small environmental changes based on majority consensus.
**Platform:** Twitch + YouTube chat
**Frequency:** Continuous, with 5-minute voting windows
**Restriction:** Subscriber-only + 1 vote per user per window

| Mechanic | Detail |
|---|---|
| Trigger | Bot announces a vote topic (e.g., "What should Truman eat?") |
| Duration | 5-minute window |
| Resolution | Majority wins |
| User limit | 1 vote per user per window |
| Visibility | Bot posts results in chat |
| Impact | Low (food choice, music genre, weather) |

**Vote categories:**
- Food selection (what appears in fridge)
- Music genre (ambient playlist selection)
- Weather (rain, sun, snow, storm)
- Small room changes (poster, plant state, lighting)

### Tier 2: Channel Points (Medium Impact)

**Purpose:** Individual viewer-triggered events with immediate effect.
**Platform:** Twitch Channel Points
**Frequency:** Instant, with per-type cooldown
**Restriction:** 10-minute cooldown per redemption type per user

| Redemption | Cost (Points) | Effect | Cooldown |
|---|---|---|---|
| Change weather | 500 | Weather shifts for 30 minutes | 10 min |
| Doorbell ring | 1,000 | Mysterious doorbell, Truman investigates | 10 min |
| Gift a book | 750 | New book appears on shelf | 10 min |
| Strange noise | 500 | Unexplained sound, Truman reacts | 10 min |
| New food delivery | 500 | Food item appears in fridge | 10 min |
| Power flicker | 1,500 | Lights briefly flicker, subtle awakening trigger | 10 min |

**Channel Points are naturally scarce**, providing built-in rate limiting. Viewers earn points by watching, which encourages sustained viewership.

### Tier 3: Hourly Polls (High Impact)

**Purpose:** Major activity suggestions decided by community vote.
**Platform:** Twitch Polls (native feature)
**Frequency:** Every 60 minutes
**Duration:** 3-minute voting window

| Mechanic | Detail |
|---|---|
| Trigger | Automated every 60 minutes |
| Options | 3-4 activity suggestions generated by the system |
| Duration | 3-minute window |
| Resolution | Most votes wins |
| Impact | High (Truman's next activity block) |
| Resistance | Truman may still refuse based on mood |

**Poll options are generated by the system**, not viewers, to prevent injection. The system proposes contextually relevant next activities based on:
- Current time of day
- Recent activity history (avoid repetition)
- Truman's current mood and energy
- Available objects and resources

## 4. Indirect Communication

### 4.1 Letters

Viewers can collectively submit content that becomes a "letter" delivered under Truman's door.

**Process:**
1. Viewers submit messages via chat command (`!letter`)
2. Messages are aggregated over 24 hours
3. AI sanitizer filters and rewrites into a coherent letter
4. Operator reviews and approves
5. Letter appears under door the next day
6. Truman reads it and reacts

**Frequency:** Maximum 1 letter per day.
**Content:** Aggregated viewer sentiment, rewritten to sound like it came from an unknown source.

### 4.2 Newspaper

A periodic "newspaper" appears with headlines influenced by viewer nominations.

**Process:**
1. Viewers nominate headlines via chat command (`!headline`)
2. Nominations aggregated weekly
3. AI generates a newspaper page from approved nominations
4. Operator reviews
5. Newspaper appears on Truman's desk

**Frequency:** 1-2 times per week.

## 5. Interaction Flow (Technical)

```
Viewer chat/points/poll
        |
        v
    Platform API (Twitch EventSub / YouTube Chat API)
        |
        v
    Chat Ingestion Service
        |
        v
    Command Parser (identifies vote, redemption, letter, etc.)
        |
        v
    Rate Limiter (per-user, per-type cooldowns)
        |
        v
    Sanitizer Pipeline (3-layer, see security-spec.md)
        |
        v
    Vote Aggregator (collects votes for active windows)
        |     or
        v
    Event Queue (BullMQ: immediate Channel Point events)
        |
        v
    Environment Manager (applies change to world state)
        |
        v
    Agent Brain Observation System (Truman "notices" the change)
        |
        v
    Agent Brain decides reaction (may refuse based on mood)
        |
        v
    Action + speech/thought sent to Renderer
```

### 5.1 Platform Integration

| Platform | Library | Connection Type |
|---|---|---|
| Twitch Chat | @twurple/chat | WebSocket (persistent) |
| Twitch EventSub | @twurple/eventsub-ws | WebSocket (persistent) |
| Twitch API | @twurple/api | REST (on-demand) |
| YouTube Live Chat | googleapis | Polling (respecting pollingIntervalMillis) |

### 5.2 Unified Event Model

Both platforms emit events normalized to a common format:

```typescript
interface ViewerEvent {
  id: string;
  platform: 'twitch' | 'youtube';
  type: 'chat_vote' | 'channel_points' | 'poll_vote' | 'letter' | 'headline';
  userId: string;         // Platform-specific user ID
  displayName: string;    // For logging only, not stored long-term
  content: string;        // The vote choice, redemption name, or text
  metadata: Record<string, unknown>;
  timestamp: Date;
}
```

## 6. Anti-Abuse Measures

### 6.1 Access Requirements

| Feature | Requirement |
|---|---|
| Chat votes | Subscriber-only |
| Channel Points | Subscriber-only (Twitch native) |
| Polls | Open to all (Twitch native, harder to abuse) |
| Letters | Subscriber + account age > 7 days |
| Headlines | Subscriber + account age > 7 days |

### 6.2 Rate Limits

| Action | Limit |
|---|---|
| Chat votes per user per window | 1 |
| Channel Point redemptions per type | 1 per 10 minutes |
| Letter submissions per user per day | 1 |
| Headline nominations per user per week | 3 |
| Chat commands per user | 1 per 30 seconds |

### 6.3 Duplicate Detection

- Same user voting multiple times in a window: only first vote counts
- Same content submitted by multiple users: deduplicated, counts as +1 weight
- Identical letter content from different users: merged

### 6.4 Escalation

If abuse is detected (coordinated spam, bot accounts, targeted harassment):
1. Switch to subscriber-only mode
2. Increase account age requirement
3. Enable Twitch/YouTube slow mode
4. Operator alerted via Discord
5. If sustained, temporarily disable affected interaction tier

## 7. Chaos Mode

**Frequency:** Weekly, Saturday prime time (exact time TBD based on viewer analytics)
**Duration:** 1 hour
**Promotion:** Announced 24 hours in advance via stream overlay and social media

### 7.1 Chaos Mode Changes

| Mechanic | Normal | Chaos Mode |
|---|---|---|
| Vote windows | 5 minutes | 2 minutes |
| Poll frequency | Every 60 minutes | Every 15 minutes |
| Poll options | 3-4 | 5-6 (wilder options included) |
| Channel Point cooldown | 10 minutes | 3 minutes |
| Truman refusal rate | ~30% | ~15% (more compliant) |
| Event types | Standard | Expanded (surprise gifts, room rearrangement, mystery visitor sounds) |

### 7.2 Chaos Mode Safety

All safety bounds still apply during chaos mode:
- Sanitizer pipeline active
- Emotional floor enforced
- Content rating remains PG-13
- Cost caps still enforced (chaos mode has its own budget allocation)

## 8. YouTube-Specific Considerations

### 8.1 API Quota Budget

YouTube Live Chat API has a daily quota of 10,000 units (default):

| Operation | Cost | Daily Budget |
|---|---|---|
| Read chat messages | ~5 units | ~1,500 reads/day |
| Send bot message | ~50 units | ~100 sends/day |
| Moderation action | ~50 units | As needed |

### 8.2 Polling Strategy

- Use `pollingIntervalMillis` from API response (typically 5-10 seconds)
- Batch process messages received in each poll
- Reduce bot message frequency to conserve write quota
- Consider requesting quota increase before launch

### 8.3 Feature Parity

YouTube lacks Channel Points and native Polls. Alternatives:

| Twitch Feature | YouTube Alternative |
|---|---|
| Channel Points | Super Chat donations (not directly equivalent) |
| Polls | Bot-driven chat polls (count !vote1, !vote2) |
| EventSub | Manual polling (no push API) |

Maintain feature parity where possible, accept limitations where not.
