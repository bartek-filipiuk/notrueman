# Asset Pipeline — No True Man Show

How to generate, modify, and manage AI-generated pixel art assets.

## Overview

```
Prompt (text) → Retro Diffusion (Replicate API) → PNG → public/sprites/ → Phaser BootScene
```

All assets are 16-bit pixel art generated by **Retro Diffusion** models via the Replicate API. The game loads PNGs at runtime with programmatic fallback if files are missing.

## Quick Start

```bash
# Set API token (get from https://replicate.com)
export REPLICATE_API_TOKEN=r8_xxx

# Generate all assets (~30 images, ~$0.90)
./scripts/generate-assets.sh

# Verify output
ls packages/renderer/public/sprites/objects/
ls packages/renderer/public/sprites/truman/
```

| Model | ID | Use Case | Output | Cost |
|-------|----|----------|--------|------|
| **rd-plus** | `retro-diffusion/rd-plus` | Static sprites (objects, poses) | Single PNG, transparent bg | ~$0.03 |
| **rd-tile** | `retro-diffusion/rd-tile` | Seamless tiles | Tileable PNG | ~$0.03 |
| **rd-animation** | `retro-diffusion/rd-animation` | Spritesheets | Multi-frame strip | ~$0.05 |

## Prompt Guidelines

### General
- Always include: `pixel art`, `side view`
- Color guidance: `warm colors`, `cozy style`
- Background: use `remove_bg: true` for transparent backgrounds
- Resolution: match intended in-game size (e.g., 128x64 for bed)

### Characters
```
chibi pixel art character, young man [action/emotion], blue shirt, dark pants,
brown messy hair, large head small body, cute proportions, game sprite, facing right
```

### Room Objects
```
[object description], side view, pixel art room furniture, warm cozy style
```

### Tiles
```
[material] texture, pixel art seamless texture, [style keywords]
```

## Adding a New Asset

### 1. Add generate command to `scripts/generate-assets.sh`

```bash
generate "retro-diffusion/rd-plus" \
  "your prompt here, pixel art, side view, warm colors" \
  WIDTH HEIGHT "$OUT_DIR/objects/new_item.png"
```

### 2. Add programmatic fallback

In `packages/renderer/src/sprites/RoomObjectSprites.ts`:

```typescript
function generateNewItem(scene: Phaser.Scene): void {
  const g = scene.add.graphics();
  // Draw fallback shape...
  g.generateTexture("obj_new_item", WIDTH, HEIGHT);
  g.destroy();
}
```

### 3. Register in BootScene

The `BootScene.preload()` loads from `sprites/objects/` automatically for known keys. If adding a completely new object, add the preload call.

### 4. Add to ROOM_OBJECTS

In `packages/shared/src/constants.ts`, add the object definition with position, size, and interaction properties.

## Regenerating Assets

To regenerate a single asset, edit the prompt in `scripts/generate-assets.sh` and run just that `generate()` call. The script is idempotent — it overwrites existing files.

To regenerate all assets:
```bash
REPLICATE_API_TOKEN=r8_xxx ./scripts/generate-assets.sh
```

## Post-Processing

Current pipeline does not include post-processing. If needed:
- **Color quantization**: `pngquant --quality=65-80 sprite.png` (reduce to 16-32 colors)
- **Resize**: `convert sprite.png -filter Point -resize 200% sprite_2x.png` (nearest-neighbor)
- **Trim transparency**: `convert sprite.png -trim +repage sprite.png`

## Troubleshooting

### "REPLICATE_API_TOKEN not set"
Set the token: `export REPLICATE_API_TOKEN=r8_xxx`

### Generation times out
Replicate cold-starts can take 30-60s. The script polls for 60s before timing out. Retry the failed generation.

### Image looks wrong
1. Adjust the prompt — be more specific about style, angle, colors
2. Try different dimensions (some objects render better at different sizes)
3. Regenerate — AI output varies between runs

### Missing PNG in game
BootScene falls back to programmatic `generateTexture()`. Check browser console for preload errors.
